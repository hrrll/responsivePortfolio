Responsive Portfolio Website[1] - Setting Up Workflow and Structure

This project uses VS Code with three extensions
liveserver by Ritwick Dey and
Bracket Pair Colorizer by CoenraadS and
Prettier - Code Formatter by Esben Petersen. To have it format automatically, go into settings, select the gear icon in lower left screen, and enter "editor.formatOnSave": true,. In the search bar if you type "editor." the intellisense should bring up the proper setting. Change setting from false to true and hit enter and the code will automatically be added to settings or you can just type it into the settings file. Then save file.
Also need nodejs, a Javascript runtime, because it comes with npm, node package manager. The compile tool used with Sass is called Node Sass and npm is required to install it.
Also recommend using Git, which is version control.

  1. Create / select directory
  2. Create folder
  3. Create a dist folder. It will contain all html, css, images, javascript, etc.
  4. Create index.html file
  5. Start Live Server by right clicking in html file and selecting to Open with Live Server
  6. Create folder outside of dist called scss. This folder will contain sass files.
  7. Within this folder, create a file call main.scss. Browsers don't read .scss files so to compile the files down to css install node.sass. It uses libSass which is the "C" version of the preprocessor.
    1. create a pkg.json file which is a manifest for project by
      1. Using either the standard Windows terminal or use the integrated terminal in VS Code (access by right clicking), enter npm init and answer questions. This will create the package.json file which will appear in the file structure. Responses for this project are:
        • package name: (modern_portfolio)
        • version: (1.0.0)
        • description: Responsive Portfolio Website
        • entry point: (index.js)
        • test command: hit enter
        • git repository: hit enter
        • keywords: hit enter
        • author: "Enter your name"
        • license: "MIT:
        • HIT ENTER
    2. Go back to terminal window and type npm i node-sass. This will create a node_modules folder. It is not deployed to server and you never really have to touch anything in folder and there is a lot because node-sass is a dependency and some of its dependencies have dependencies.
    3. Now to set up node-sass to work using a npm script.
      • In the package.json file, locate the "scripts" object. It will say something like "scripts": { "test": "echo \"Error: no test specified\" && exit 1"
        change to: "scripts": { "sass": "echo \"Hello from sass". This could be changed to say anything desired.
      • Save file
      • Test was changed to sass, this is the script name. To run the script, go back to the terminal and type npm run
      • What we really want here is to run node-sass in watch mode to watch the scss folder and to then output the compiled sass to dist/css which contains regular css. To do this change the scripts object to read: "scripts": { "sass": "node-sass -w scss/ -o dist/css/ --recursive".The CSS folder under dist directory will automatically be created.
      • Save file
      • To run the script, go back to the terminal and type npm run code npm run sass
  8. Now go to the main.scss and write some sass.
  9. To see that the css folder is automatically created, in the main.scss file type $primary-color: #444; body { background: $primary-color;}

    Adding the $ in front of primary-color indicated primary-color is a variable

    In the terminal window a message will indicate its saving to the .css file and a css folder will be created. The sass was compiled to regular css and saved in the css folder in a file call main.css. DON'T TOUCH THE REGULAR CSS FILE!

  10. Go to index.html and add a link to main.css
  11. Create a git repository
    • Open a new terminal by hitting the plus sign. At this point, there are two terminal windows: One running sass and the other git.
    • type touch .gitignore. The touch command creates a new file
    • Locate the gitignore file in the file directory.
    • Type in the gitignore file, node_modules
    • Save and close file
    • Initialize repository by typing in the terminal git init. This creates a hidden folder called git. All repository stuff goes in there.
    • To add all the project work completed so far, in the terminal window type git add . to add all.
    • To make a local commit to the repository with a comment, type in the terminal window: git commit -m 'Initial workflow setup'. Where Initial workflow setup is the comment.

If you look in the developer tools under the computed properties it will say 96px. If the html property designated the font-size as 10px, 6REM would equal 60 px.
